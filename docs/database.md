# Структура базы данных

## Общая информация

Для хранения данных в проекте используется система управления базами данных SQLite.

Выбор SQLite обусловлен следующими преимуществами:

* простота интеграции с Python;
* отсутствие необходимости отдельного серверного ПО;
* низкие системные требования;
* удобство использования для небольших и средних проектов;
* высокая скорость работы при локальном хранении данных.

База данных используется для хранения информации о пользователях, заявках на помощь, координатах и статусах заявок.

## Назначение базы данных

База данных обеспечивает:

* хранение информации о пользователях;
* сохранение пользовательских заявок;
* обработку информации о местоположении;
* хранение статусов заявок;
* быстрый доступ к данным системы.

Использование базы данных позволяет обеспечить постоянное хранение информации между сеансами работы чат-бота.

## Основные таблицы базы данных

### Таблица users

Таблица предназначена для хранения информации о зарегистрированных пользователях системы.

Основные поля таблицы:

| Поле              | Назначение                            |
| ----------------- | ------------------------------------- |
| id                | Уникальный идентификатор пользователя |
| vk_id             | Идентификатор пользователя ВКонтакте  |
| username          | Имя пользователя                      |
| registration_date | Дата регистрации                      |

Данная таблица используется для идентификации пользователей и связи пользователей с созданными заявками.

---

### Таблица requests

Таблица содержит заявки на взаимопомощь, создаваемые пользователями.

Основные поля таблицы:

| Поле        | Назначение                      |
| ----------- | ------------------------------- |
| id          | Уникальный идентификатор заявки |
| user_id     | Идентификатор автора заявки     |
| description | Текстовое описание проблемы     |
| category    | Категория помощи                |
| latitude    | Географическая широта           |
| longitude   | Географическая долгота          |
| status      | Текущий статус заявки           |
| created_at  | Дата создания заявки            |

Статусы заявок позволяют отслеживать текущее состояние запроса.

Примеры статусов:

* активна;
* выполняется;
* завершена.

## Связи между таблицами

Таблица requests связана с таблицей users через поле user_id.

Один пользователь может создавать несколько заявок, что соответствует связи типа «один ко многим».

## Принцип работы базы данных

При взаимодействии пользователя с ботом система выполняет следующие действия:

1. Получает данные от пользователя.
2. Проверяет корректность введенной информации.
3. Сохраняет данные в базе данных.
4. Выполняет поиск и фильтрацию заявок.
5. Возвращает результат пользователю.

Все операции выполняются автоматически через модуль работы с базой данных.

## Работа с геолокацией

Для реализации поиска ближайших заявок в базе данных сохраняются географические координаты.

Используются следующие параметры:

* широта (latitude);
* долгота (longitude).

Координаты позволяют реализовать отображение заявок на карте и поиск заявок в определенном радиусе.

## Обеспечение целостности данных

Для предотвращения ошибок используются следующие меры:

* проверка корректности пользовательского ввода;
* контроль существования пользователя;
* использование уникальных идентификаторов;
* ограничение некорректных значений.

## Преимущества выбранной структуры

Выбранная структура базы данных обладает следующими преимуществами:

* простота реализации;

* возможность масштабирования;
