# Архитектура системы

## Общая информация

Проект представляет собой чат-бота для социальной сети ВКонтакте, предназначенного для координации локальной взаимопомощи между пользователями с использованием интерактивного картографического сервиса.

Система позволяет пользователям размещать заявки на помощь, указывать свое местоположение и получать информацию о ближайших запросах других пользователей.

Основной задачей архитектуры является обеспечение стабильной обработки пользовательских запросов, хранения данных и взаимодействия с внешними сервисами.


# Основные компоненты системы

Система состоит из нескольких взаимосвязанных компонентов:

* пользовательский интерфейс ВКонтакте;
* модуль взаимодействия с VK API;
* модуль обработки команд;
* модуль работы с базой данных;
* модуль обработки геолокации.

# Пользовательский уровень

Пользователь взаимодействует с системой через чат сообщества ВКонтакте.

Через интерфейс мессенджера пользователь может:

* запускать бота;
* создавать заявки;
* просматривать доступные запросы;
* отправлять геолокацию;
* получать уведомления и ответы.

Передача данных между пользователем и системой осуществляется посредством VK API.

# Модуль взаимодействия с VK API

Данный модуль отвечает за:

* получение сообщений пользователей;
* обработку входящих событий;
* отправку ответов;
* обработку команд;
* получение информации о пользователях.

После получения события от VK API бот определяет тип запроса пользователя и передает его в систему обработки команд.

Основные функции модуля:

* прием пользовательских сообщений;
* идентификация пользователя;
* отправка текстовых сообщений;
* обработка кнопок и команд;
* обработка геолокации.

# Модуль обработки команд

Модуль обработки команд является центральной частью системы.

Он отвечает за:

* анализ пользовательского ввода;
* определение необходимого сценария работы;
* запуск соответствующих функций системы.

В рамках данного модуля реализуются сценарии:

* регистрация пользователя;
* создание заявки;
* поиск заявок;
* просмотр информации;
* изменение статуса заявки.

После обработки команды система либо обращается к базе данных, либо формирует ответ пользователю.

# Модуль базы данных

Для хранения информации используется база данных SQLite.

База данных содержит информацию:

* о пользователях;
* о заявках;
* о координатах;
* о статусах заявок.

Модуль работы с базой данных отвечает за:

* подключение к базе данных;
* выполнение SQL-запросов;
* сохранение данных;
* получение информации;
* обновление записей.

# Модуль геолокации

Система поддерживает обработку географических координат пользователей.

Геолокация используется для:

* отображения заявок на карте;
* поиска ближайших запросов;
* фильтрации заявок по местоположению.

Пользователь может передавать свое местоположение через интерфейс ВКонтакте.

Полученные координаты сохраняются в базе данных и используются при поиске информации.

# Схема взаимодействия компонентов

Работа системы осуществляется по следующему сценарию:

1. Пользователь отправляет сообщение боту ВКонтакте.
2. VK API передает событие приложению.
3. Модуль обработки команд анализирует запрос.
4. Система выполняет необходимую бизнес-логику.
5. При необходимости выполняется обращение к базе данных.
6. Формируется ответ пользователю.
7. Ответ отправляется через VK API.

Упрощенная схема взаимодействия:

Пользователь → VK API → Чат-бот → Бизнес-логика → База данных
